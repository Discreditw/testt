# Static School UI (for GitHub Pages)

This repository contains a static frontend for a school site with:
- Schedule page
- Grades page (client-side filtering, static data)
- Chat page (UI only, messages are local and not sent to any server)
- Admin page showing DB tables (interface only)
- Login page (visual only)

## How to publish to GitHub Pages
1. Create a new GitHub repository.
2. Copy all files from this archive to the repository root.
3. Commit and push to branch `main`.
4. In repo Settings -> Pages, enable GitHub Pages for branch `main` (root).
5. Open your site at `https://USERNAME.github.io/REPO_NAME/`.

## Notes
- This project is intentionally static. There is **no backend**, no real authentication and no real database.
- Use it to demonstrate UI and workflows. For training with real vulnerabilities use OWASP Juice Shop or a local isolated lab.
